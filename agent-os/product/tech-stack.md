# Tech Stack

## Overview

Argus is a Kotlin Multiplatform library that embeds a Ktor HTTP+WebSocket server inside a host application, captures Ktor `HttpClient` traffic and KMMLogging events, and serves a static TypeScript SPA (bundled in-process as byte arrays) over the LAN for browser-based inspection. JVM and Android targets ship in Phase 1; iOS lands in Phase 4 with zero re-architecture. Distribution is **debug-only** — consumers wire Argus via `debugImplementation`, and release builds contain no Argus code at all.

## Language & Build

- **Kotlin 2.x** across every module.
- **Gradle** with Kotlin DSL; all versions and plugin aliases live in `gradle/libs.versions.toml` (matches `kmp/module-build-conventions`). Root `build.gradle.kts` declares plugins with `apply false` for consistent cross-module versioning.
- **Package root:** `com.lynxal.argus` — subpackages follow `naming/package-structure` (singular folder names, one top-level declaration per file).
- **JVM target:** 17 (aligned with `kmp/module-build-conventions`).
- **Android min SDK:** **24.** *Divergence from `kmp/module-build-conventions` (which sets minSdk = 26).* Argus is a debug-only library consumed by third-party Canvas/Lynxal apps, so it needs a broader min SDK reach than the Canvas Control app itself.
- **Versioning:** **Semver (`x.y.z`).** *Divergence from `kmp/version-management` (which uses `{major}.{minor}.{buildId}` with `System.currentTimeMillis()`).* Argus is distributed as a library via the internal Lynxal Maven repo; consumers pin and range on semver. Wire compatibility is tracked separately as `schemaVersion` in the WebSocket hello payload.
- **Publishing:** internal Lynxal Maven repo.

| Module | Type | Targets | Purpose | Key Dependencies |
| --- | --- | --- | --- | --- |
| `:argus-core` | KMP | jvm, android, ios | Ktor client plugin, KMMLogging delegate, event schema, event bus. | `kotlinx.coroutines`, `kotlinx.serialization`, Ktor client core, `com.lynxal.logging:logging` |

| `:argus-android` | Android library | android | Public Android entry point — `Argus.start(context)`, `ArgusHandle`, `AppInfoBuilder`, `LocalIp`, `AndroidArgusDriverFactory`. | `:argus-server-core`, `:argus-core`, sqldelight-android-driver |
| `:argus-ios` | KMP (iOS-only) | ios | Public iOS entry point — `Argus.start { … }`, `ArgusHandle`, `AppInfoBuilder` (NSBundle/UIDevice), `LocalIp` (getifaddrs), `IosArgusDriverFactory`. | `:argus-server-core`, `:argus-core`, sqldelight-native-driver |

### Dependency flow (one-way, per `kmp/module-boundaries`)

```
:argus-android ─┐             ┌── :argus-ios
                ├─ :argus-server-core ──┬─ :argus-core
                │                       └─ :argus-webui-bundle ── :argus-webui (npm)
                └── (no mDNS module — URL exposed via ArgusHandle.url)
```

No module ever depends upward. `:argus-core` sits at the base with no internal dependencies; `:argus-webui` is a sibling with no Kotlin sources and is only consumed indirectly through the bundle module.

## Core (KMP) — `:argus-core`

- **Kotlin 2.x**
- **`kotlinx.coroutines` 1.8+** — suspending capture APIs and the internal event bus.
- **`kotlinx.serialization` JSON** — event schema serialization.
- **Ktor client core + plugin API** — capture via `createClientPlugin`; hooks cover full request/response including bodies.
- **`com.lynxal.logging:logging` (KMMLogging)** — log delegate that forwards application log events into the Argus event stream.

Targets: `jvm`, `androidTarget`, `iosArm64`, `iosSimulatorArm64`, `iosX64`.

## Server-Core (KMP) — `:argus-server-core`

- **Ktor server core** — REST endpoints + WebSocket stream.
- **Depends on** `:argus-core` (events) and `:argus-webui-bundle` (static SPA assets).
- **`ArgusServer` is a plain `commonMain` class.** `io.ktor.server.cio` is multiplatform (JVM + Native), so no `expect/actual` split is required. The original spec proposed host-module actuals; the implementation collapsed them once it became clear both platforms produce byte-identical bodies.
- **`ArgusConfigBuilder`** also lives in `commonMain` and is shared by both `:argus-android` and `:argus-ios` via the public `argusConfig(appInfo, configure)` bridge.
- Server logic (routing, ring buffer, query/filter) lives in `commonMain/` so both platforms share it.

## Android — `:argus-android`

- **Android min SDK:** 24.
- **Engine:** Ktor Server **CIO** (JVM, via `:argus-server-core`).
- **Discovery:** none — `ArgusHandle.url` exposes the bound URL via `LocalIp.firstIPv4()` (NetworkInterface scan). mDNS/Bonjour is a future-work item.
- **Entry point:** `Argus.start(context) { … }` — public Android-only API that host apps use to start/stop Argus (wired via `debugImplementation` only — see Distribution).

## iOS — `:argus-ios`

- **Engine:** Ktor Server **CIO** (Native, via `:argus-server-core`).
- **Persistence:** SqlDelight `NativeSqliteDriver` (host app must add `OTHER_LDFLAGS = -lsqlite3`).
- **Discovery:** none — `ArgusHandle.url` exposes the bound URL via `LocalIp.firstIPv4()` using POSIX `getifaddrs` (from `platform.darwin`).
- **Entry point:** `Argus.start { … }` — Swift-friendly Kotlin object exported via the iOS framework.
- **Out of scope:** URLSession capture for non-Ktor iOS apps. Hosts must use Ktor `HttpClient`.

## Web UI — `:argus-webui`

- **Vanilla TypeScript + Vite.**
- **Tailwind CSS.**
- **`@preact/signals-core`** for reactive state.
- **Design tokens + component specs** are sourced from the Claude Design output in `design_handoff_argus_inspector/`:
  - `design_handoff_argus_inspector/ds/` — `colors_and_type.css`, fonts, logo mark.
  - `design_handoff_argus_inspector/argus/` — component references (`Inspector.jsx`, `EventList.jsx`, `EventDetail.jsx`, `FilterBar.jsx`, `TopBar.jsx`, `Waterfall.jsx`, `BodyViewer.jsx`, `Overlays.jsx`, `Primitives.jsx`) used as specs for the TS components.
- **Bundle size — soft target:** gzipped bundle < **100 KB**. Treated as a design budget, not a build-fail ceiling. If the target is breached, trim deps or split before adding a gate.

The SPA is a standalone npm project. It has no Gradle presence except through a thin wrapper task (`:argus-webui:npmBuild`) that `:argus-webui-bundle:generateBundle` depends on.

## webui-bundle generation — `:argus-webui-bundle`

A Gradle task `generateBundle` produces the embedded assets. Steps:

1. Depends on `:argus-webui:npmBuild`.
2. Reads `../argus-webui/dist/`.
3. gzip-compresses each file.
4. Emits `src/commonMain/generated/ArgusUiBundle.kt` containing a base64 payload + content-type map.

`compileKotlin` of `:argus-webui-bundle` depends on `generateBundle`, so any consumer that builds the module automatically regenerates assets if the SPA dist changed.

At runtime there is no filesystem dependency — the SPA is pure byte constants compiled into the KMP library.

## Build conventions

- **Gradle Kotlin DSL.** `settings.gradle.kts` enables `TYPESAFE_PROJECT_ACCESSORS` and explicitly `include`s each module.
- **Version catalog** (`gradle/libs.versions.toml`): every dependency version + plugin alias. Modules consume via `alias(libs.plugins.kotlinMultiplatform)`, `libs.ktor.server.core`, etc.
- **Root `build.gradle.kts`:** plugins declared with `apply false` for consistent versioning (per `kmp/module-build-conventions`).
- **iOS framework:** static by default (`isStatic = useStaticFramework`, default `true`); override with `-PuseStaticFramework=false`. Each module's framework `baseName` matches the module name (`argus-core`, `argus-server-core`, `argus-webui-bundle`, `argus-ios`).
- **Compiler opt-ins:** `freeCompilerArgs.add("-opt-in=kotlin.time.ExperimentalTime")` on all modules; additional opt-ins per module as needed.
- **Publishing:** internal Lynxal Maven repo.

## Distribution

- **Debug-only.** Consumers wire Argus via `debugImplementation` (and optionally a custom `stagingImplementation`).
- **`releaseImplementation` is explicitly unsupported.** Release builds contain zero Argus code.
- **No no-op module by design.** Release builds don't link against Argus at all — no shim, no stub, no dead weight. This is a deliberate choice, not an oversight.

## Applicable standards (quick reference)

- **`agent-os/standards/kmp/module-boundaries.md`** — Argus's module graph is strictly one-way; `:argus-core` is the base and no module depends upward.
- **`agent-os/standards/kmp/module-build-conventions.md`** — plugin assignment, version catalog, iOS static framework default, JVM 17. *Diverges on Android minSdk.*
- **`agent-os/standards/kmp/expect-actual-conventions.md`** — followed in spirit (*"Keep expect declarations minimal — push shared logic to commonMain"*); `ArgusServer` is now a plain `commonMain` class because Ktor server CIO is multiplatform and no per-platform state is required.
- **`agent-os/standards/kmp/version-management.md`** — cited for contrast; Argus uses semver rather than a `currentTimeMillis()` buildId.
- **`agent-os/standards/naming/package-structure.md`** — package root `com.lynxal.argus`; singular folder names; one top-level declaration per file.
